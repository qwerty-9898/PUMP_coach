## 1. Что это
**PUMP** — фитнес-приложение «онлайн-тренер» для новичков в зале (рынок СНГ).
Формат: **Telegram Mini App** (PWA на React). Уклон — **в тренировки и онлайн-тренера**, подсчёт калорий вторичен.

**Зачем Telegram, а не сторы:** нет комиссий 15–30%, нет риска отказа Apple (правило 4.2), быстрый запуск и шеринг. Это основной канал продвижения и монетизации.

**Монетизация (план):** бесплатное ядро + платная подписка (расширенная аналитика, ИИ-фишки, счётчик калорий по фото — в разработке).

**Главное отличие от конкурентов:** связка «тренировки (план + гид + прогрессия) + питание» в одном, заточенная под новичка и под русский язык. Конкурент по калориям @vkusai_bot — ранняя стадия, только фото-калории.

## 2. Стиль и тематика
- **Тёмная спорт-неон тема**, премиальный вид.
- Цвета (CSS-переменные в `styles.css`):
  - фон `--bg #0b0d10`, карточки `--card #161a22` / `--card2 #1d222c`, линии `--line #262c38`
  - акцент `--accent #ff5a1f` (оранж), `--accent2 #ff8a3d`, подложка `--accent-soft rgba(255,90,31,.14)`
  - текст `--text #f1f4f9`, приглушённый `--muted #98a2b3`
- Цвета групп мышц: грудь `#ff5a1f`, спина `#3b82f6`, ноги `#22c55e`, плечи `#a855f7`, бицепс `#ec4899`, трицепс `#f59e0b`, пресс `#14b8a6`.
- **Шрифты (актуальные):** `Rubik` — текст (округлый, дружелюбный), `Unbounded` — заголовки и крупные цифры (модный жирный). *(Раньше были Oswald/Manrope — заменены полностью.)*
- Иконки — только **линейные SVG** (`Icon.jsx`), без эмодзи/смайликов.
- **Единая система карточек:** класс `.card` (градиент `165deg, card2→card`, скругление 20px, мягкая тень). Шапки блоков — `.card-head` + `.card-kicker` (иконка+CAPS) + `.card-meta`.

## 3. Технологии и workflow
- **React 18 + Vite**, single-page (роутинг через state, без router-либы).
- Хранилище: **localStorage** через `src/storage.js`, ключи с префиксом `pump_`.
- Telegram SDK: `telegram-web-app.js` в `index.html`, обвязка в `src/tg.js`.
- Графики/иконки тела — **самописный SVG** (`SparkChart.jsx`, `BodyIcon.jsx`, `BodyHeatmap.jsx`). Внешние chart-либы НЕ используем (npm в песочнице заблокирован; CDN-либы работают только в виджет-инструменте, не в приложении).
- **Деплой:** Vercel + GitHub (репозиторий `qwerty-9898/PUMP_coach`) + BotFather Mini App. Есть `vercel.json`, `.gitignore`, `.env`/`.env.example`, `DEPLOY_TELEGRAM.md`.

## 4. Структура (src/)

### Движки (чистый JS, тестируемые)
- `engine/exercises.js` — `GROUPS` (7), `GROUP_META` (label/short/color), `levelGE`, **`EXERCISES` (~95 упр)**: `{id,name,group,equip:'none'|'dumbbell'|'gym',compound,level,muscles,technique[],mistakes[],tip}`, `techExtras(ex)`. Счёт: грудь 15, спина 14, ноги 19, плечи 12, бицепс 9, трицепс 11, пресс 15.
- `engine/sessionGenerator.js` — `generateSession({groups,goal,level,equip,favorites,varied})`, `pickAlternative(...)`, `shapeExercise(...)`. Учитывает избранное и инвентарь.
- `engine/nutrition.js` — `calcNutrition` (Mifflin-St Jeor) → `{bmr,tdee,kcal,protein,fat,carbs}`.
- `engine/loads.js` — `estimateLoad(ex,profile)` → «≈ X кг» / «кг/рука» / «вес тела».
- `engine/progress.js` — `e1rm`, `exSeries(exId)`, `personalRecord`, `allRecords`, `weeklyVolume`, `suggestNext` (авто-прогрессия: +2.5кг при верхе диапазона, иначе +1 повтор), `lastSummary`, `isPR`.
- `engine/recovery.js` — **карта восстановления**: `recoveryMap()` (свежесть группы по последней дате, load 1=сегодня…0=давно, окно 4 дня, `statusOf`), `coverageMap(days,target)` (как часто грузил группу за N дней — для дашборда), `freshFocus(limit)` (что качать сегодня).
- `engine/achievements.js` — **медали**: `achievements()` (10 бейджей: первый шаг, в деле, серия 3/7, 10/25 тренировок, всё тело, рекордсмен×5, 10т/50т тоннаж; каждый `{earned, progress}`), `achievementsSummary()`.

### Данные
- `data/programs.js` — `PROGRAMS` (6: Фуллбади, Верх/Низ, PPL, 2 сплита синергист/антагонист, бро-сплит), у каждой `equip:[]`. `programsForEquip`, `recommendProgram({level,daysPerWeek,equip})` (учёт уровня И инвентаря), `buildWeek` (`DAY_PATTERNS`).
- `data/foods.js` — ~89 продуктов СНГ, `searchFoods`, `macrosFor`.

### storage.js
Ключи `K`: profile `pump_profile_v3`, progress, water, measures, food, fav, favex, rating, logs `pump_logs_v1`.
Методы: профиль (get/set/clear/`clearAll`=стереть все `pump_`), `getProgress/setProgress` (`workouts[]`), дневник еды, избранное (еда+упр `getFavEx/toggleFavEx`), рейтинг, **логи** (`addLog/getExLogs/getLastSets/loggedExIds`), `todayKey()`, `calcStreak(dates)`.

### Компоненты (главные)
- `App.jsx` — корень: profile/route/loading state. Нижняя таб-навигация `TABS=[home,workout,catalog,progress,more]`; вторичные экраны `{profile,nutrition,water,measures,timer,calculators}`; Loader ~2.6с после онбординга; Telegram BackButton.
- `Home.jsx` — **новая главная**: приветствие → **карта восстановления** (центральный блок `BodyHeatmap` + список групп со статусом + подсказка «Сегодня свежее всего») → герокарта «Начать тренировку» → `CalendarStrip` → сводка дня (серия/ккал/вода) → плитки (Каталог/Инструменты) → совет дня.
- `BodyHeatmap.jsx` — **X-ray силуэт** (череп/позвоночник/рёбра/таз/кости, светящиеся суставы оранжем, viewBox 48×80), фронт+спина, мышцы-зоны со свечением по `load`. Принимает `loads` и `onPick`.
- `BodyIcon.jsx` — мини X-ray скелет с подсветкой зоны группы в её цвете (каталог).
- `Progress.jsx` — **дашборд**: герой-карта (серия + за 7д/тренировок/тоннаж) → **медали** (сетка, earned горят / locked с %) → **карта «Прокачано за 30 дней»** (`coverageMap`) → личные рекорды → график роста веса (`SparkChart`) → тоннаж по неделям → оценка программы (звёзды) → любимые упражнения → история.
- `More.jsx` — **шапка профиля** (кружок-инициал + имя + уровень/цель → в профиль) + **секции списком**: «Тело и питание» (Калории, Вода, Замеры), «Инструменты» (Таймер, Калькуляторы, Каталог), «Система» (Перезагрузить приложение, Стереть все данные — перенесены сюда из Профиля).
- `Workout.jsx` / `ProgramPicker.jsx` / `WorkoutBuilder.jsx` — выбор программы (фильтр по инвентарю), пресеты, «Собери за меня», ручной выбор групп, генерация сессии, «Гид по шагам», «Другой вариант», замена упражнения, смена программы.
- `GuidedWorkout.jsx` — пошаговый режим: разминка→работа→финиш. По упражнению: прошлый результат + подсказка прогрессии, ввод вес/повторы по подходам, лог в `logs`, вспышка при PR, таймер отдыха, сохранение тренировки.
- `Catalog.jsx` — группы (`GroupBadge`+счёт) → список → деталь (demo-карта с большим `BodyIcon` + техника/ошибки/темп-дыхание-разминка).
- `Nutrition.jsx` — «живой» счётчик: кольцо ккал, бары БЖУ, приёмы пищи, поиск еды, ручной ввод, избранное; тизер «счётчик по фото».
- Прочее: `CalendarStrip.jsx`, `WeekSchedule.jsx`, `Water.jsx`, `Measures.jsx`, `Timer.jsx`, `Calculators.jsx`, `Onboarding.jsx` (пошаговый визард), `Profile.jsx` (быстрый редактор + «Пройти знакомство заново»; системные кнопки убраны в «Ещё»), `Loader.jsx`, `SparkChart.jsx`, `BodyIcon.jsx`, `GroupBadge.jsx`, `Icon.jsx`.

## 5. Что уже сделано (ключевое)
- Полный ребрендинг: спорт-неон, единая система карточек `.card`, шрифты Rubik+Unbounded.
- Главная пересобрана вокруг **карты восстановления мышц** (идея от Fitbod/Hevy).
- **Прогресс** = дашборд с **медалями/ачивками** и картой покрытия за 30 дней.
- **«Ещё»** = шапка профиля + секции списком; системные кнопки (перезагрузка/сброс) перенесены сюда из Профиля.
- Логирование подходов → история → личные рекорды → графики → **авто-прогрессия** (главная «душа» приложения: цикл обратной связи).
- Каталог ~95 упражнений с техникой; фильтрация программ/упражнений по инвентарю; рекомендация под уровень.
