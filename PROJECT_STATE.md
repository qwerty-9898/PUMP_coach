## 1. Что это
**PUMP** — фитнес-приложение «онлайн-тренер» для новичков в зале (рынок СНГ).
Формат: **Telegram Mini App** (PWA на React). Уклон — **в тренировки и онлайн-тренера**, подсчёт калорий вторичен.

**Зачем Telegram, а не сторы:** нет комиссий 15–30%, нет риска отказа Apple (правило 4.2), быстрый запуск и шеринг. Это основной канал продвижения и монетизации.

**Монетизация (план):** бесплатное ядро + платная подписка (расширенная аналитика, ИИ-фишки, счётчик калорий по фото — в разработке).

**Главное отличие от конкурентов:** связка «тренировки (план + гид + прогрессия) + питание» в одном, заточенная под новичка и под русский язык. Конкурент по калориям @vkusai_bot — ранняя стадия, только фото-калории.

Глобальный слой в конце `styles.css`: фон body со свечением (radial оранж сверху, синий снизу, `fixed`); поверхности (`.card` и др. контейнеры) — `--glass rgba(19,23,31,.55)` + `backdrop-filter: blur(14px)` + светлая окантовка `--glassbd`; таб-бар/топбар — frosted blur. Оранжевые герои (`.herocard/.profilecard/.dash-hero/.welcome/.tip2`) и активные состояния (`.on`) НЕ тронуты. Скругление карточек 22px.

## 3. Технологии и workflow
- **React 18 + Vite**, single-page (роутинг через state, без router-либы).
- Хранилище: **localStorage** через `src/storage.js`, ключи с префиксом `pump_`.
- Telegram SDK: `telegram-web-app.js` в `index.html`, обвязка в `src/tg.js`.
- Графики/иконки тела — **самописный SVG** (`SparkChart.jsx`, `BodyIcon.jsx`, `BodyHeatmap.jsx`). Внешние chart-либы НЕ используем (npm в песочнице заблокирован; CDN-либы работают только в виджет-инструменте, не в приложении).
- **Деплой:** Vercel + GitHub (репозиторий `qwerty-9898/PUMP_coach`) + BotFather Mini App. Есть `vercel.json`, `.gitignore`, `.env`/`.env.example`, `DEPLOY_TELEGRAM.md`.

### Правила работы с файлами (ВАЖНО)
- Инструмент Write/Edit **режет кириллицу** на длинных файлах (NUL-байты). Поэтому файлы пишем через **bash heredoc** (`cat > file << 'EOF'`) или python, мелкие правки — `sed`.
- После каждой правки: проверка **баланса скобок** (python-счётчик `{} () []`) + `node --check` для `.js` (JSX через node не чекается — только баланс скобок + ревью).
- Логику движков тестируем **node ESM-тестом** с шимом `localStorage`.

## 4. Структура (src/)

### Движки (чистый JS, тестируемые)
- `engine/exercises.js` — `GROUPS` (7), `GROUP_META` (label/short/color), `levelGE`, **`EXERCISES` (~95 упр)**: `{id,name,group,equip:'none'|'dumbbell'|'gym',compound,level,muscles,technique[],mistakes[],tip}`, `techExtras(ex)`. Счёт: грудь 15, спина 14, ноги 19, плечи 12, бицепс 9, трицепс 11, пресс 15.
- `engine/sessionGenerator.js` — `generateSession({groups,goal,level,equip,favorites,varied})`, `pickAlternative(...)`, `shapeExercise(...)`. Учитывает избранное и инвентарь.
- `engine/nutrition.js` — `calcNutrition` (Mifflin-St Jeor) → `{bmr,tdee,kcal,protein,fat,carbs}`.
- `engine/loads.js` — `estimateLoad(ex,profile)` → «≈ X кг» / «кг/рука» / «вес тела».
- `engine/progress.js` — `e1rm`, `exSeries(exId)`, `personalRecord`, `allRecords`, `weeklyVolume`, `suggestNext` (авто-прогрессия: +2.5кг при верхе диапазона, иначе +1 повтор), `lastSummary`, `isPR`.
- `engine/recovery.js` — **карта восстановления**: `recoveryMap()` (свежесть группы по последней дате, load 1=сегодня…0=давно, окно 4 дня, `statusOf`), `coverageMap(days,target)` (как часто грузил группу за N дней — для дашборда), `freshFocus(limit)` (что качать сегодня).
- `engine/achievements.js` — **медали** (9, без дублей механик, категории Регулярность/Объём/Сила): первый шаг, неделя огня (серия 7), полная неделя (4 за 7д), 30/100 тренировок, 100 подходов, 25 тонн, личный рекорд, +10 кг к старту. `achievementsSummary()`.

### Данные
- `data/programs.js` — `PROGRAMS` (6: Фуллбади, Верх/Низ, PPL, 2 сплита синергист/антагонист, бро-сплит), у каждой `equip:[]`. `programsForEquip`, `recommendProgram({level,daysPerWeek,equip})` (учёт уровня И инвентаря), `buildWeek` (`DAY_PATTERNS`), `activeOrRecommended(profile,activeId)`.
- `data/foods.js` — **174 продукта СНГ** (10 категорий), `searchFoods`, `macrosFor`.

### storage.js
Ключи `K`: profile `pump_profile_v3`, progress, water, measures, food, fav, favex, rating, logs `pump_logs_v1`.
Методы: профиль (get/set/clear/`clearAll`=стереть все `pump_`), `getProgress/setProgress` (`workouts[]`), дневник еды, избранное (еда+упр `getFavEx/toggleFavEx`), рейтинг, **логи**, питание 2.0 (freq/dishes/kcalgoal/foodWeek/updateFood), **активная программа** (`getActiveProgram/setActiveProgram`), `todayKey()`, `calcStreak(dates)`.

### Компоненты (главные)
- `App.jsx` — корень: profile/route/loading/`catGroup` state. Нижняя таб-навигация (`NavIcon` — duotone, заливка при активной) `TABS=[home,workout,catalog,progress,more]`; `openMuscle(g)` → каталог группы; вторичные экраны `{profile,nutrition,water,measures,timer,calculators}`; Loader ~2.6с после онбординга; Telegram BackButton.
- `Home.jsx` — **новая главная**: приветствие → **карта восстановления** (центральный блок `BodyHeatmap` + список групп со статусом + подсказка «Сегодня свежее всего») → герокарта «Начать тренировку» → `CalendarStrip` → сводка дня (серия/ккал/вода) → плитки (Каталог/Инструменты) → совет дня.
- `BodyHeatmap.jsx` — **большая карта тела на мышечной анатомии** (PNG `assets/skeleton/muscle_front|back.png`), переключатель Спереди/Сзади, яркая заливка мышц по `load`. Зоны в `bodyZones.js` (%). Каталог — на скелет-кропах (`g_*.png`) с красной неон-подсветкой целевой мышцы (`MUSCLE_HL`).
- `muscleArt.js` — `MUSCLE_ART` (группа→скелет-кроп) + `MUSCLE_HL` (красная подсветка зоны). `MenuIcon.jsx` — кастомные duotone-иконки для «Ещё». (`BodyIcon.jsx` удалён.)
- `Progress.jsx` — **дашборд**: герой-карта (серия + за 7д/тренировок/тоннаж) → **медали** (сетка, earned горят / locked с %) → **карта «Прокачано за 30 дней»** (`coverageMap`) → личные рекорды → график роста веса (`SparkChart`) → тоннаж по неделям → оценка программы (звёзды) → любимые упражнения → история.
- `More.jsx` — **премиум-шапка профиля** (аватар + имя + уровень/цель + статы серия/тренировок/поднято) + секции с **кастомными duotone-иконками** (`MenuIcon`): Тело и питание, Инструменты, Система (перезагрузка/сброс).
- `Workout.jsx` / `ProgramPicker.jsx` / `WorkoutBuilder.jsx` — выбор программы (фильтр по инвентарю), пресеты, «Собери за меня», ручной выбор групп, генерация сессии, «Гид по шагам», «Другой вариант», замена упражнения, смена программы.
- `GuidedWorkout.jsx` — пошаговый режим: разминка→работа→финиш. По упражнению: прошлый результат + подсказка прогрессии, ввод вес/повторы по подходам, лог в `logs`, вспышка при PR, таймер отдыха, сохранение тренировки.
- `Catalog.jsx` — группы (`GroupBadge`+счёт) → список → деталь (demo-карта с большим `BodyIcon` + техника/ошибки/темп-дыхание-разминка).
- `Nutrition.jsx` — счётчик 2.0: кольцо + **редактируемая цель ккал** (авто под цель / ручная), бары БЖУ, **график калорий за 7 дней**, приёмы пищи с **правкой порции** (шит) и удалением, AddPanel со **вкладками** (Поиск/Частые/Избранное/Мои блюда/Вручную), **конструктор своих блюд** (`DishBuilder`). Частые считаются по `frequentFoods`.
- Прочее: `CalendarStrip.jsx`, `WeekSchedule.jsx`, `Water.jsx`, `Measures.jsx`, `Timer.jsx`, `Calculators.jsx`, `Onboarding.jsx` (пошаговый визард), `Profile.jsx` (быстрый редактор + «Пройти знакомство заново»; системные кнопки убраны в «Ещё»), `Loader.jsx`, `SparkChart.jsx`, `BodyIcon.jsx`, `GroupBadge.jsx`, `Icon.jsx`.

## 8. Ребрендинг v1.4 (по пунктам пользователя)
- **Главная**: карта тела — фон-сцена (`.bodystage`), без коробки-карточки; крупный body (330px) + плавающие стеклянные чипы статусов групп (`.bs-chip`) + «свежее всего». Как у Fitbod.
- **Каталог**: картинки 7 групп перенарезаны из **мышечной анатомии** (`generated-image (3).png` → `g_*.png`), не из скелета; красная подсветка зоны (`MUSCLE_HL`).
- **Тренировки** (`ProgramPicker`): рекомендованная программа — крупный оранжевый герой с фоном-телом (`.prog-hero`), остальные — компактный frosted-список с раскрытием (`.progrow`).
- **Прогресс**: дашборд с вкладками (`.nut-tabs`): Обзор / Графики / Медали / История. Герой-карта сверху со статами и шерингом.
- **«Ещё» → «Профиль»**: таб переименован, иконка навигации `more` = силуэт пользователя; экран — профиль-герой (`.profilehero`: аватар, чипы уровня/цели, статы, «Изменить профиль») + секции инструментов.
